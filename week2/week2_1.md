객체지향의 사실과 오해
=====================
# 객체   
식별자, 상태와 행동을 함께 지닌 실체   
객체는 다른 객체와 협력하기 위해 메시지를 전송   
객체가 수신된 메시지를 처리하는 방법을 메서드   
메서드는 객체의 자율   
     ->메시지와 메서드를 분리함으로써 자율성을 높임   
     

<hr/>

## 상태   
- 특정 시점에 객체가 가지고 있는 정보의 집합으로 객체의 구조적 특징을 표현   
- 객체에 존재하는 정적인 프로퍼티와 동적인 프로퍼티 값으로 구성   
### 객체의 프로퍼티는 단순한 값과 다른 객체를 참조하는 링크로 구분   

## 행동   
- 외부의 요청 또는 수신된 메시지에 응답하기 위해 동작하는 활동

## 쿼리   
- 객체의 상태를 조회하는 작업

## 명령   
- 객체의 상태를 변경하는 작업

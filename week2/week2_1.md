## 1장 협력하는 객체들의 공동체
### 협력, 그리고 역할, 책임
 ***"어떤 객체도 섬이 아니다[Beck 1989]"***  
 * 객체는 다른 객체와 **협력** 하는데, 각 정해진 **역할**을 수행하며 역할은 관련된 **책임**의 집합이다.
   1. 여러 객체가 동일한 역할을 수행할 수 있다.
   2. 역할은 대체 가능성을 의미한다.
   3. 각 객체는 책임 수행 방법을 자율적으로 선택할 수 있다.
   4. 하나의 객체가 동시에 여러 역할을 수행할 수 있다.
 * 객체는 상태와 행위를 가진 자율적인 존재다.  
   **객체지향**은 시스템을 상호작용하는 **자율적인 객체들의 공동체**로 바라보고 객체를 이용해 시스템을 분할하는 방법을 말한다.
 * 협력하기 위해 수신된 메시지를 처리하는 방법을 **메서드**라고 한다.
### 객체를 지향하라
 * 클래스는 협력에 참여하는 객체를 만드는 데 필요한 구현 메커니즘일 뿐이다.
 * 어떤 객체들이 어떤 메시지를 주고받으며 협력하는가에 집중해야 한다.

## 2장 이상한 나라의 객체
### 객체의 상태, 행동, 식별자
 * 객체는 상태를 가지며 상태는 변경 가능하다.
 * 객체의 **상태를 변경**시키는 것은 객체의 **행동**이다.
   * 행동의 결과는 상태에 의존적이며 상태를 이용해 서술할 수 있다.
   * 행동의 순서가 실행 결과에 영향을 미친다.
 * 객체는 어떤 상태에 있더라도 유일하게 **식별 가능**하다.
### 상태가 아니라 행동에 초점을 맞춰라
 ***책임-주도 설계(Responsibility-Driven Design, RDD) 하자!***   
 * 상태를 먼저 결정할 경우
 1. 캡슐화가 저해된다.
 2. 객체를 협력자가 아닌 고립된 섬으로 만든다.
 3. 객체의 재사용성이 저하된다.

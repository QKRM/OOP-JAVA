# **객체지향의 사실과 오해**  
106쪽 까지의 내용  

**"객체지향 소프트웨어는 실세계의 투영이며, 객체는 현실세계에 존재하는 사물의 추상화다"**  
라고 하는 문장은 반은 정확하고,반은 틀린 말이라고 한다.  
철학적인 정의의 객체지향에 관해서는 맞는 이야기지만 실질적인 소프트웨어의 설계 에서는 틀린 말이라는 이야기다.  

상태와 행위를 '캡슐화'하는 소프트웨어 객체의 '자율성' 을 설명하고,  
서로 '메시지'를 주고받으며 '협력'하는 객체들의 관계를 설명하는데 적합하다.

협력에는 여러 객체들이 참여한다.  
책임 - 객체가 가지는 모든 메서드  
역할 - 협력 내에서 쓰이는 책임이 역할  

![image](https://user-images.githubusercontent.com/48343739/112803149-f3dcae00-90ad-11eb-9a7a-13c0f16f986c.png)

협력 = 요청 + 응답  
* 요청이 요청을 연쇄한다  
* 요청과 응답은 반대의 방향으로 진행된다  
* 역할과 책임은 떨어질 수 없다.  

협력의 특징으로는  
* 여러사람이 동일 역할을 수행 가능하다  
* 역할 = 대체가능성 (substitude)  
* 책임을 수행하는 방법은 자율적으로 선택할 수 있다.= 동일 요청에 대해 서로 다른 방식으로 응답이 가능하다 (다형성 : polymorphism)  
* 한 사람이 동시에 여러 역할을 수행할 수 있다.  
![image](https://user-images.githubusercontent.com/48343739/112804176-0dcac080-90af-11eb-80c0-46e8f94b9caa.png)

이는 커피를 따르는 과정에서, 사장이 커피를 타는경우, 알바가 계산을하고 청소도 하는경우,  
라떼아트를 다르게 그리는 경우 등을 생각해보면 알 수 있다  

# **객체란 협력에 참여하는 주체를 의미한다.**  
이러한 객체에는 두가지 덕목이 있다고 한다.  

**객체는 협력적이어야 한다**  
다른 객체에서 오는 요청,응답을 받는다.(혼자 모든걸 해결하지 않는다)  
애초에 객체라는 개념은 혼자서 모든걸 해결하기 힘들기 때문에 분산하기 위해 만든 개념  

**객체는 자율적이어야 한다**  
음...다른 객체의 요청이나 응답을 듣고 스스로 판단해야한다는 이야기인 것 같다.  

**객체는 상태와 행동을 함께 지니고 있다.  
또 외부와  내부가 반드시! 구분된다**  
내부와 외부 이야기는 자율적어야 한다. 라는 것과 관련이 있다고 생각이 든다.

서로 요청과 응답을 하기위해서 객체 들은 메시지를 통해 의사소통하고  
메시지를 수신한 객체는 메시지에 대응되는 메서드를 실행한다.(자율성과 다형성)  

# **객체 지향의 본질**  
1. 객체 지향 : 시스템을 상호작용하는 자율적인 객체들의 공동체로 바라보고 객체를 이용해 시스템을 분별하는 방법  
2. 자율적인 객체 : 상태와 행위를 함께 지니며 스스로 자기 자신을 책임지는 객체  
3. 객체는 시스템의 행위를 구현하기 위해 다른 객체와 협력한다. 각 객체는 협력 내에서 정해진 역할을 수행하며 역할은 관련된 책임의 집합이다  
4. 객체는 다른 객체와 협력하기 위해 메시지를 전송하고 메시지를 수신한 객체는 메시지를 처리하는데 적합한 메서드를 자율적으로 선택한다
